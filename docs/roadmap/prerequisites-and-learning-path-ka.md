# წინასწარი ცოდნა და სასწავლო გზა ამ პროექტისთვის (ქართული)

ეს დოკუმენტი აღწერს, რა ტიპის ცოდნაა სასარგებლო ამ პროექტის გასაგებად
და გასაგრძელებლად. ეს არ არის „ქულა/გამოცდა“, არამედ გზა, რომელიც
ჩემთვის და სხვა დამწყებთათვის შეიძლება კომფორტული იყოს.

---

## 1. საბაზისო უნარები (არა-SDR)

SDR-ზე ჩასაღრმავებლად სასარგებლოა, თუ ცოტათი მაინც ჭრილიხარ:

- **ოპერაციული სისტემის ძირითადი გამოყენება**
  - ტერმინალით/კონსოლით მუშაობა (PowerShell, cmd, ან Linux shell),
  - პროგრამების ინსტალაცია,
  - საქაღალდეებში გადაადგილება.

- **პროგრამირების საფუძვლები**
  - ელემენტარული Python (ცვლადები, მასივები, პატარა სკრიპტები),
  - `pip`-ით ბიბლიოთეკების დაყენება.

- **მათემატიკის საფუძვლები**
  - სინუსი და კოსინუსი,
  - კომპლექსური რიცხვები (რეალური + წარმოსახვითი ნაწილი),
  - ძირითადი ინტუიცია, რომ Fourier-ის ტრანსფორმაცია „დროს“ გადაჰყავს
    „სიხშირის“ გამოსახულებაში.

სიღრმისეულ სპეციალისტობა არ არის საჭირო – იდეა ისაა, რომ ამ პროექტის
მეშვეობითაც ისწავლო.

---

## 2. SDR-ის და რადიოს საფუძვლები

HackRF-ზე და SDR-ზე მუშაობისთვის სასარგებლოა იცოდე:

- **რა არის რადიოტალღა**
  - სიხშირე, ტალღის სიგრძე,
  - როგორ უკავშირდება სიხშირე კონკრეტულ დიაპაზონებს
    (FM ბენდი, 1090 MHz და სხვ.).

- **სიგანის ზოლი და სემპლის სიხშირე**
  - ფართო ბენდი → მეტ სიხშირეს ვხედავთ, მაგრამ მეტ მონაცემს ვიღებთ,
  - SDR-ში sample rate დაახლოებით ტოლია იმ სიხშირის სიგანისა,
    რომელსაც „ვიჭერთ“.

- **IQ მონაცემები**
  - რას ნიშნავს I და Q,
  - რატომ იყენებს SDR-ის ერთ კომპლექსურ ნიმუშს და არა ერთ რეალურს
    (იხილე `docs/iq-and-spectra-basics.md` და `docs/iq-basics-ka.md`).

- **სპექტრი და ვოთერფოლი**
  - როგორ უნდა წავიკითხოთ,
  - რა განსხვავებული „ნახატი“ აქვს FM-ს, ADS-B-ს და ხმაურს.

---

## 3. პროგრამული ხელსაწყოები

ამ პროექტში ვგეგმავ გამოვიყენო:

- **Python**
  - `numpy`, `matplotlib` – IQ მონაცემების დასამუშავებლად და გრაფიკების
    დასახატავად,
  - `examples/` საქაღალდეში არსებული სკრიპტები.

- **GNU Radio (შემდგომ ეტაპზე)**
  - უფრო რთული ფლოოუგრაფებისთვის (FM, ADS-B, თანამგზავრები).

- **SDR მიმღერები**
  - GQRX / SDRangel და მსგავსები:
    - სპექტრის და ვოთერფოლის სწრაფი ნახვა,
    - ანტენის/სიხშირის კონფიგურაციის შემოწმება.

ყველაფრის ერთად ცოდნა თავიდან აუცილებელი არ არის – იდეა ისაა, რომ
პირველ რიგში Python-ით და მარტივი ვიზუალიზაციით დავიწყო, შემდეგ კი
ნელ-ნელა დავამატო GNU Radio და ცოცხალი მიღება.

## 4. სასწავლო გზა ამ რეპოზიტორში

ნაცადი გზა, რომელსაც მე თვითონ გავივლი და სხვასაც შეიძლება გამოადგეს:

1. **IQ და სპექტრი – ძირითადი იდეები**
   - წაიკითხე: `docs/iq-and-spectra-basics.md` (ინგლ.) და
     `docs/iq-basics-ka.md` (ქარ.).
   - სცადე: `examples/generate_test_tone.py`,
     `plot_spectrum.py`, `plot_waterfall.py`.
   - მიზანი: გაიგო, რა არის IQ და როგორ იღება სპექტრი/ვოთერფოლი.

2. **სიგნალის ჯაჭვის გაგება**
   - წაიკითხე: `docs/signal-chain-overview.md` (ინგლ.) და
     `docs/signal-chain-ka.md` (ქარ.).
   - მიზანი: დიდი სურათი – ანტენა → HackRF → IQ → პროგრამა →
     გრაფიკები/აუდიო/დატა.

3. **FM რადიო**
   - წაიკითხე: `docs/fm-from-iq-basics.md` (ინგლ.) და
     `docs/fm-from-iq-basics-ka.md` (ქარ.).
   - მოგვიანებით სცადე: `examples/fm_demo_from_file.py` რეალურ FM IQ ფაილზე.
   - მიზანი: analog მოდულაციის მაგალითი, IQ-დან პირველ „ხმის“ მიღება.

4. **ADS-B – თვითმფრინავების სიგნალები**
   - წაიკითხე: `docs/adsb-from-iq-basics.md` (ინგლ.) და
     `docs/adsb-from-iq-basics-ka.md` (ქარ.).
   - შემდეგ ეტაპზე: არსებული ADS-B IQ ჩანაწერების ან/და დექოდერების
     გამოყენება თვითმფრინავების პოზიციების სანახავად.
   - მიზანი: ციფრულ, პულსურ სიგნალზე გადასვლა (1090 MHz) და ნახო,
     როგორ ხდება რეალური მონაცემების ამოღება IQ-დან.

5. **თანამგზავრები (მომავალი ეტაპი)**
   - მომავალში დავამატებ ამინდის/ამატორი თანამგზავრების მიღების გეგმებს,
     როცა ამის შესაძლებლობა და კანონიერება იქნება.
   - მიზანი: IQ, FFT, დემოდულაცია და დეკოდირება გადმოვიყვანო ვიზუალურად
     საინტერესო რაღაცაში (დედამიწის სურათები, ტელემეტრია და ა.შ.).

---

## 5. იურიდიული და ეთიკური ჩარჩო

ძალიან მნიშვნელოვანია:

- ვიმუშაო მხოლოდ იმ სიგნალებთან, რომელთა მიღება კანონიერია ჩემი ქვეყნისთვის.
- არ გადავცე და არ „გავაღვიძე“ სიგნალები იმ დიაპაზონებში, სადაც ამის
  უფლება არ მაქვს.
- ამ პროექტს მივუდგე როგორც **სასწავლო ლაბორატორიას**, და არა როგორც
  „ჰაკინგ“ ინსტრუმენტს.

---

## 6. გზის განახლება

ეს სასწავლო გზა საბოლოო არ არის:

- როცა მეტს ვისწავლი, ამ დოკუმენტსაც შევცვლი.
- შეიძლება დავამატო დამატებითი ნაბიჯები (მაგ. მარტივი AM დემოდულაცია,
  FSK მაგალითები),
- ან შევცვალო თანმიმდევრობა, თუ უკეთესი ახსნის გზა ვიპოვე.

თუ შენც დამწყები ხარ და რაიმე იდეა გაქვს, როგორ გავხადოთ ეს ახსნა
უკეთესი, შენიშვნები/პულ რექვესტები მივესალმებით.
